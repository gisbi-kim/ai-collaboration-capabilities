# AI Collaboration Capabilities

This repository hosts a Korean essay-style guide titled **"AI 시대의 4가지 진짜 역량"**.

The page argues that the real gap in the AI era is not simple access to tools, but the quality of a person's thinking loop: how they ask, verify, reconstruct, and monitor their own understanding while working with AI.

## Live Page

https://gisbi-kim.github.io/ai-collaboration-capabilities/

## Core Message

AI can make answers easier to obtain, but it does not automatically improve judgment. The document frames AI collaboration as a cycle of four capabilities:

1. **Questioning**: defining the real problem, context, constraints, success criteria, and failure conditions.
2. **Verification**: checking whether an AI-generated answer is factually, logically, contextually, and strategically reliable.
3. **Reconstruction**: turning AI outputs into structures that fit one's own purpose, such as hypotheses, experiments, proposals, papers, or implementation plans.
4. **Metacognition**: noticing the difference between having an answer and actually understanding it.

## What The Page Contains

- A high-level explanation of why AI acts more like a thinking amplifier than a simple equalizer.
- A comparison table between weak and strong AI collaboration habits.
- Examples of bad questions and better questions.
- A verification framework for checking AI-generated outputs.
- A reconstruction framework for turning raw AI responses into useful artifacts.
- A metacognition section about avoiding the illusion of understanding.
- Individual and team training routines.
- A simple rubric for evaluating AI collaboration capability.
